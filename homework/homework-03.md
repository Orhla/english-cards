## Часть 1: Скрипт как на проде

Открой `scripts/generate-audio.ts`. Там несколько проблем — исправь каждую.

### 1.1 Бинарники не хранятся в репозитории

`.ogg` файлы попали в git. Это неправильно — бинарные файлы не должны жить в репозитории.

Удали их из репо и добавь в `.gitignore`:
```
public/audio/
```

После этого запусти скрипт заново и убедись что файлы генерируются.

### 1.2 Ранняя проверка ключа

Если `YANDEX_API_KEY` не задан в `.env` — скрипт сейчас молча идёт дальше и падает с непонятной ошибкой где-то внутри. Это плохо.

Добавь в самое начало скрипта проверку: если ключ не задан — бросай ошибку сразу с понятным сообщением, до любой работы с базой и файлами.

### 1.3 Guard clause

Сейчас код выглядит примерно так:
```ts
if (!existsSync(audioPath)) {
    // 40 строк логики
}
```

Это называется "вложенность" — и это усложняет чтение. Перепиши через guard clause:
```ts
if (existsSync(audioPath)) {
    continue; // уже есть — пропускаем
}
// дальше логика без вложенности
```

Разница небольшая, но код читается лучше.

### 1.4 Чистка при ошибке

Сейчас если запрос к Яндексу упал на середине — на диске остаётся битый файл. При следующем запуске скрипт его найдёт, пропустит, и карточка будет воспроизводить мусор.

Исправь: если при записи файла произошла ошибка — удали его перед тем как пробросить ошибку дальше.

Подсказка: `fs/promises` умеет удалять файлы — посмотри там.

### 1.5 Ретраи с backoff

Внешние API иногда возвращают ошибки временно — перегрузка, сеть, rate limit. Просто упасть в этом случае — неправильно.

Напиши функцию `withRetry` которая:
- принимает асинхронную функцию и количество попыток
- при ошибке ждёт перед следующей попыткой
- каждый раз ждёт **вдвое дольше** (первый раз 1с, потом 2с, потом 4с...)
- если все попытки исчерпаны — бросает ошибку

```ts
async function withRetry<T>(fn: () => Promise<T>, attempts: number): Promise<T> {
    // твоя реализация
}
```

Оберни в неё вызов к Яндекс API. `setTimeout` который был раньше — убери.

### 1.6 Параллельность с ограничением

Сейчас карточки обрабатываются одна за другой. Если их 1000 — скрипт будет работать очень долго.

Надо запускать несколько запросов параллельно — но не все сразу (это перегрузит API). Оптимально — не более 5 одновременно (должно быть настраиваемо).

> Существуют готовые библиотеки для этого, например `p-limit`. Но сначала попробуй реализовать сама через `Promise.all` — это полезно понять изнутри.
вот хорошее объяснение: https://maxgreenwald.me/blog/async-pool-promise.all-for-bulk-operations

вот еще, но первое лучше:

## Часть 2: Заменяем распознавание речи

Самописный `src/lib/speech-recognition.ts` работает, но у него есть проблемы — мы их разбирали на уроке. Вместо того чтобы чинить самописный код, используем проверенную библиотеку.

Установи:
```bash
npm install react-speech-recognition
npm install -D @types/react-speech-recognition
```

Документация: https://github.com/JamesBrill/react-speech-recognition

### 2.1 Перепиши CardViewer

Замени использование `RussianMeaningSpeechRecognition` на хук из библиотеки.

Библиотека даёт тебе:
- `listening` — булево, идёт ли сейчас запись
- `transcript` — что распознали
- `SpeechRecognition.startListening({ language: '...' })` — начать запись
- `SpeechRecognition.stopListening()` — остановить

Обрати внимание: `listening` из библиотеки — это именно то состояние которого не хватало в прошлой версии. Теперь кнопка микрофона должна визуально отражать что запись идёт.

### 2.2 Останови запись при смене карточки

В `changeCard()` добавь остановку записи — вдруг пользователь переключился пока говорил.

`SpeechRecognition` из библиотеки доступен напрямую через импорт — ref здесь не нужен.

### 2.3 useRef для аудио

Кнопка динамика сейчас выглядит так:
```ts
new Audio(`/audio/${card.word}.ogg`).play()
```

Объект `Audio` создаётся и сразу теряется — остановить его невозможно. Если переключить карточку пока играет звук — он будет играть в фоне.

`useRef` — это способ хранить значение между рендерами **без** вызова ре-рендера. Подходит для "технических" объектов: таймеры, WebSocket, медиа-элементы.

Исправь:
- сохраняй объект `Audio` в ref
- при вызове `changeCard()` — останавливай предыдущее аудио если играет

```ts
const audioRef = useRef<HTMLAudioElement | null>(null);
```

### 2.4 Удали старый файл

`src/lib/speech-recognition.ts` больше не нужен — удали его.

### 3.2 Настраиваемый CardViewer

Два режима отличаются поведением:

| | Главная `/` | Практика `/practice` |
|---|---|---|
| Лицевая сторона | английское слово | английское слово |
| Микрофон слушает | русский перевод (`ru-RU`) | английское произношение (`en-US`) |
| Сравниваем с | `card.translation` | `card.word` |

Добавь в `CardViewer` проп `mode`:
```ts
type Mode = 'translation' | 'pronunciation'

type Props = {
    cards: WordCard[]
    mode: Mode
}
```

Внутри компонента поведение меняется в зависимости от `mode` — язык распознавания и то с чем сравниваем транскрипт.

Главная передаёт `mode="translation"`, страница практики — `mode="pronunciation"`.
