## Пререквизит — Prisma адаптер

Сейчас при логине через Яндекс Auth.js создаёт JWT токен, но пользователь **не сохраняется в базу**. Модель `User` в схеме есть, но она пустая — никто туда ничего не пишет.

Чтобы это исправить, нужен **Prisma адаптер** — это прослойка которая говорит Auth.js "при логине сохраняй пользователя вот в эту базу".

Установи:

```bash
npm install @auth/prisma-adapter
```

Подключи в `auth.ts`:

```ts
import { PrismaAdapter } from "@auth/prisma-adapter"
import { prisma } from "@/lib/prisma" // или где у тебя живёт Prisma-клиент
```

Добавь адаптер и стратегию сессий в конфиг:

```ts
export const { handlers, signIn, signOut, auth } = NextAuth({
    adapter: PrismaAdapter(prisma),
    session: { strategy: "jwt" },  // важно — без этого Auth.js переключится на DB-сессии
    providers: [...],
})
```

Почему `strategy: "jwt"` обязательно? По умолчанию, когда адаптер есть, Auth.js переходит на хранение сессий в таблице `Session` в базе. Мы этого не хотим — мы хотим токены.
Эта строчка явно говорит "используй JWT несмотря на адаптер".

Проверь: залогинься, открой Prisma Studio — в таблице `User` должна появиться запись.

### 1.2 Записать роль в JWT токен

Сейчас Auth.js хранит в токене стандартный набор полей: `id`, `email`, `name`, `image`. Роль там не появится сама по себе — нам нужно добавить её вручную через **callbacks**.

Открой `auth.ts` в корне проекта.

Auth.js при создании токена вызывает callback `jwt` — туда можно дописать любые поля. При каждом запросе сессии вызывается callback `session` — туда нужно пробросить поля из токена, чтобы они были доступны в коде.

Структура:

```ts
export const { handlers, signIn, signOut, auth } = NextAuth({
    providers: [...],
    callbacks: {
        async jwt({ token, user }) {
            // user существует только при первом логине
            // token — это то, что запишется в JWT
            // здесь нужно: если user есть, сходить в базу и дописать role в token
        },
        async session({ session, token }) {
            // session — это то, что вернёт auth() в компонентах
            // здесь нужно: переложить role из token в session.user
        }
    }
})
```

В callback `jwt` тебе понадобится Prisma-клиент чтобы достать роль пользователя из базы по его `id`. Найди `id` в объекте `user` (он есть при первом логине) и сделай запрос.

**Важный момент:** TypeScript будет ругаться что поля `role` нет в типах `token` и `session.user` — это стандартное поведение. Auth.js позволяет расширить типы через module augmentation. Погугли "next-auth typescript extend session type" — это несложно и полезно знать.

Проверь: после логина вызови `auth()` в любом серверном компоненте и выведи `session` в консоль — там должна быть роль.
